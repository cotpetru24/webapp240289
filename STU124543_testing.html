Task 3 : Testing

Once the website is ready, the next part of web authoring is testing. To ensure that website meets all the standards, is compatible with as many browsers as possible and can be accessed on as many devices as possible, the following tests were performed, and to eliminate all the errors website was modified accordingly.

  • w3c Html Validation

    • - /index.html was tested using Html validator and it identified that <!DOCTYPE html> was missing and trailing slash was pressent on many void elements.
      - To eliminate the errors, warnings and unnecessary syntax, <!DOCTYPE html> was added at the beginning of html document and trailing slash was removed from void elements.
      - Performing Html validation test after html file was corrected, confirmed that /index.html contains no errors, warnings or unnecessary syntax.

    • - /discover.html was tested using Html validator and it identified that <!DOCTYPE html> was missing, a stray end tag for a div element was pressent,the frameborder attribute on the iframe element was pressent but it is obsolete, and trailing slash was pressent on many void elements.
      - To eliminate the errors, warnings and unnecessary syntax, <!DOCTYPE html> was added at the beginning of html document,stray end tag for a div element was deleted, the frameborder attribute on the iframe element was moved from html file to css file and replaced with border attribute,and trailing slash was removed from void elements.
      - Performing Html validation test after html file was corrected, confirmed that /discover.html contains no errors, warnings or unnecessary syntax.

  • Browser compatibility

    To confirm that website is compatible with different browsers /index.html and /activities.html were tested in Chrome, Edge and Firefox.

  • Devices compatibility

    Using Chrome Developer Tools, /index.html and /activities.html were tested to confirm that website displays correctly on different mobile devices.
    As well, to make it easier for users to use the website from mobile devices, when the width of the device is less that specified in the /main.css file, page layout slightly changes.
    Top navigation bar transforms in a menu, and sidebar transforms in a menu.

    • Accessibility testing

      Chrome browser and Wave extension were used to test Accessibility.

    • Performing Accessibility test on /index.html identified some errors, missing form label, empty links and missing alternative text for images.
      To correct the errors, missing form label was added, empty links were deleted and alternative text for pictures was added. As well testing confirmed that there are no contrast errors.
